
# Google Docs Integration Guide

## Overview

Google Docs is used for collaborative documents. This guide covers how to work with Google Docs in your PM workflow.

## Usage

### Linking to Google Docs

Reference Google Docs in your Cursor docs:

```markdown
## Related Documents
- Strategy Doc: [Google Doc](https://docs.google.com/document/d/doc-id)
- Research Notes: [Google Doc](https://docs.google.com/document/d/doc-id)
- Meeting Notes: [Google Doc](https://docs.google.com/document/d/doc-id)
```

### Syncing Content

While Cursor can't directly sync with Google Docs, you can:

1. **Copy content**: Copy from Google Docs to Cursor docs
2. **Reference links**: Link to Google Docs in your docs
3. **Export**: Export Google Docs as Markdown and import to Cursor

### In PRDs

Reference Google Docs:

```markdown
## Context & Problem
- User Research: [Google Doc Research](link)
- Competitive Analysis: [Google Doc Analysis](link)
```

### In Meeting Notes

Link to Google Docs:

```markdown
## Related Docs
- Previous Meeting: [Google Doc](link)
- Action Items: [Google Doc](link)
```

## Best Practices

1. **Link, don't duplicate**: Reference Google Docs rather than copying content
2. **Use consistent structure**: Align Cursor doc structure with Google Docs
3. **Export when needed**: Export Google Docs as Markdown for Cursor analysis
4. **Keep in sync**: Update both Cursor and Google Docs when content changes
